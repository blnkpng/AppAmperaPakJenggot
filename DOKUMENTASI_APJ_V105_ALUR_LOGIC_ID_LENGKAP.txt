6. ALUR LOGIKA STOK GLOBAL
====================================================================

- Timestamp
- Tanggal

- ID Item
- Qty Masuk

- Ref ID

Arah stok:
MASUK / IN      = menambah stok.
KELUAR / OUT    = mengurangi stok.
ADJUST          = koreksi stok opname.

Lokasi:
PUSAT           = gudang/dapur pusat.
OUTLET          = stok outlet.

Outlet:
ALL             = pusat / bukan outlet tertentu.
LAHOR, SENISONO, PUJON, NGUJUNG, POKOPEK = outlet tujuan.

9. ALUR MENU OUTPUT STOK
====================================================================

Halaman:
output-stok.html
JS:
assets/js/pages/apj-output-stok.js
Backend action:
getOutputInit
simpanOutputStok
getOutputPicList / getPicKaryawanList / getInventoryPicList

Tujuan:
Mencatat barang keluar dari gudang/pusat.

Jenis penggunaan:
- Pemakaian pusat.
- Operasional.
- Barang rusak/hilang.
- Transfer outlet untuk barang jual tertentu.

Logika penting:
Semua Output selalu mengurangi stok pusat.

Jika tujuan = Transfer Outlet:
- Outlet wajib dipilih.
- PIC penerima wajib/diisi.
- Untuk barang jual seperti Minuman & Barang Dagang yang terdaftar sebagai stok outlet,
  sistem menambah stok outlet juga.
- Untuk barang operasional/non-jual, sistem hanya mengurangi stok pusat dan outlet hanya dicatat sebagai informasi.

Jurnal:
1. Baris OUT dari PUSAT selalu dibuat.
2. Baris IN ke OUTLET hanya dibuat jika item memang layak masuk stok outlet.

PIC:
- Sumber resmi dari APJ_CORE_USER.USER.
- Pada V102 sempat dibuat input manual fallback.
- Pada V104/V105 sumber utama dikunci kembali ke Core User langsung.

10. ALUR MENU PREPARASI
====================================================================

Halaman:
preparasi.html
JS:
assets/js/pages/apj-preparasi.js
Backend action:
getPreparasiInit
simpanPreparasi

Tujuan:
Mencatat proses bahan mentah/bahan utama menjadi bahan siap olah atau setengah jadi.

Contoh:
- Ayam Potong 8 menjadi Ayam Kremes 8 dan Ayam Premix 8.
- Kikil mentah menjadi Kikil siap olah.
- Kentang menjadi Begedel Setengah Jadi.

Alur:
1. User pilih resep preparasi atau manual.
2. Sistem menampilkan Bahan Keluar.
3. Sistem menampilkan Hasil Masuk.
4. Qty standar dari resep boleh diedit sesuai kondisi nyata.
5. User simpan.
6. Backend menulis jurnal:
   - Bahan keluar: Modul PREPARASI, Arah KELUAR, Lokasi PUSAT.
   - Hasil masuk: Modul PREPARASI, Arah MASUK, Lokasi PUSAT.

Logika sisa/susut V100:
Tidak ada hitung sisa otomatis global.
Alasan:
- Bahan dan hasil bisa beda satuan, contoh Kg vs Pcs.
- Bahan bisa lebih dari satu.
- Hasil bisa lebih banyak angka karena beda satuan.
- Resep bumbu bisa paten tetapi tidak bisa dihitung sisa secara global.

Sisa/Susut/Rusak/Hilang:
- Dicatat manual sebagai catatan proses.
- Tidak mengubah stok otomatis.
- Stok hanya berubah dari Bahan Keluar dan Hasil Masuk.

Satuan Selisih V101:
Dropdown:
Kg, Gram, Pcs, Porsi, Pack, Box, Liter, Ml, Botol, Bungkus, Ikat, Lembar, Butir, Ekor, Loyang, Panci, Paket.
