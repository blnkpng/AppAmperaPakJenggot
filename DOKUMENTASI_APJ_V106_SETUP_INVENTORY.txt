6. LOGIKA KATEGORI
------------------------------------------------------------
Kategori adalah kelompok besar stok.

Field utama:
- Nama Kategori
- Tipe Stok
- Prefix
- Urutan
- Aktif
- Keterangan

Contoh kategori final APJ:
- Bahan Utama
- Bahan Baku/Bumbu
- Bahan Preparasi
- Produk Jadi
- Kemasan
- Minuman & Barang Dagang
- Kebersihan/Operasional

Hapus kategori memakai soft delete:
Aktif = N


7. LOGIKA ITEM
------------------------------------------------------------
Item adalah semua barang yang dihitung stoknya.

Field utama:
- Kategori
- Kode Lama
- Nama Item
- Satuan Beli
- Satuan Stok
- Satuan Produksi
- Tipe Item
- Parent/Konversi Dari
- Tampil Input
- Bisa Output
- Bisa Preparasi
- Bisa Produksi
- Bisa Transfer Produk
- Stok Minimum
- Urutan
- Aktif
- Keterangan

Hak pakai modul:
Tampil Input = item muncul di Input Stok.
Bisa Output = item bisa dikeluarkan dari gudang.
Bisa Preparasi = item bisa dipakai/menjadi hasil preparasi.
Bisa Produksi = item bisa dipakai/menjadi hasil produksi.
Bisa Transfer Produk = item bisa dikirim ke outlet.

Hapus item memakai soft delete:
Aktif = N


8. LOGIKA RESEP PRODUKSI
------------------------------------------------------------
Resep Produksi dipakai untuk proses setengah jadi/bahan menjadi produk jadi.

Header resep disimpan di MASTER_RESEP:
- ID Resep
- ID Item Hasil
- Qty Hasil Standar
- Versi
- Aktif
- Catatan
- Kode Lama

Detail bahan disimpan di MASTER_RESEP_DETAIL:
- ID Detail
- ID Resep
- ID Bahan
- Qty Bahan
- Jenis Bahan
- Catatan

Satu resep produksi bisa memiliki banyak bahan.

Contoh:
Resep: Begedel Goreng
Hasil: Begedel Goreng 10 pcs
Bahan:
- Begedel Setengah Jadi 10 pcs
- Telur 4 pcs

Catatan:
Jumlah resep adalah standar. Pada transaksi Produksi, jumlah real tetap boleh disesuaikan.


9. LOGIKA RESEP PREPARASI
------------------------------------------------------------
Resep Preparasi dipakai untuk proses bahan mentah menjadi bahan siap olah.

Sheet: RESEP_PREPARASI

Field:
- ID Resep Preparasi
- ID Item Hasil
- Qty Hasil
- ID Bahan
- Qty Bahan
- Yield %
- Aktif
- Catatan

Contoh:
Ayam Potong -> Ayam Kremes / Ayam Premix
Kentang + bumbu -> Begedel Setengah Jadi
Kikil mentah -> Kikil siap olah

Catatan penting:
Sisa/susut/rusak/hilang tidak dihitung otomatis di master resep.
Sisa/susut dicatat saat transaksi Preparasi karena kondisi fisik bisa berubah.


10. KENAPA HAPUS TIDAK DELETE PERMANEN?
------------------------------------------------------------
Karena item/kategori/resep bisa sudah dipakai di JURNAL_STOK.
Kalau baris dihapus permanen, histori stok bisa rusak dan laporan lama bisa kehilangan referensi.

Maka tombol Hapus di Setup Inventory melakukan soft delete:
- Data dibuat nonaktif.
- Riwayat tetap aman.


11. MASUKAN STRUKTUR SETUP UNTUK APJ
------------------------------------------------------------
Setup Inventory sebaiknya hanya untuk master data, bukan transaksi.

Isi paling tepat:
