ALUR LOGIKA
1. User membuka riwayat-inventory.html.
2. Halaman cek session login dari localStorage APJ_SESSION_ACTIVE dan APJ_SESSION_TOKEN.
3. Filter default adalah tanggal awal bulan berjalan sampai hari ini.
4. Frontend mengirim request ke action getRiwayatTransaksi.
5. Backend membaca JURNAL_STOK, mapping ID Item ke MASTER_ITEM, lalu mengembalikan data legacy: jenis, modul, arah, idBarang, namaBarang, kategori, qtyMasuk, qtyKeluar, qtyAdjust, satuan, petugas, keterangan.
6. Frontend menampilkan kartu ringkasan, badge jenis transaksi, dan tabel audit.
7. Filter Arah dan Lokasi/Outlet diterapkan lokal di browser agar tidak perlu request ulang.
8. Export CSV mengambil data yang sedang tampil.
9. Cetak Rekap mencetak maksimal 800 baris pertama agar A4 tetap aman. Arsip penuh disarankan pakai CSV.

FILTER HALAMAN
- Tanggal Dari
- Tanggal Sampai
- Jenis Transaksi: Semua, Input, Output, Preparasi, Produksi, Transfer, Opname
- Arah Stok: IN, OUT, ADJ
- Lokasi / Outlet
- Kata Kunci
- Limit data: 500, 1500, 3000, 5000

ATURAN AUDIT
- Jurnal bukan tempat edit transaksi.
- Jika ada kesalahan stok, buat transaksi koreksi atau Stok Opname.
- Jangan hapus baris JURNAL_STOK secara manual karena akan merusak histori.
